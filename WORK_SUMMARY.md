# 작업 요약

## 목표

설계변경 내역서 엑셀 파일을 자동으로 변환하는 Windows용 Python GUI 프로그램을 만들었다. 사용자가 엑셀 파일과 시트를 선택하면 `당초/변경` 열을 삽입하고, 금액 수식이 있는 원본 행 아래에 `변경` 행을 추가한다.

## 구현 내용

- `app.py`
  - `tkinter` 기반 GUI를 구현했다.
  - 파일 선택, 시트 선택, 삽입 기준 열 선택, 자동화 실행 버튼을 제공한다.
  - 파일 읽기, 열 자동탐색, 변환 실행을 백그라운드 스레드로 처리해 GUI가 `응답 없음`으로 멈추지 않게 했다.
  - 처리 중 움직이는 상태 바를 추가했다.

- `excel_change_processor.py`
  - `openpyxl` 기반 엑셀 변환 로직을 구현했다.
  - `내역서`, `내역` 시트를 우선 선택할 수 있게 했다.
  - 단일 헤더와 2단 헤더 구조를 모두 인식한다.
  - `노무비`, `재료비`, `경비`, `금액` 열을 자동 탐색한다.
  - 원본 파일은 그대로 두고 `원본파일명_설계변경자동화.xlsx` 결과 파일을 만든다.
  - 깨진 인쇄 제목 정의(`#N/A`) 때문에 `openpyxl`이 파일을 열지 못하는 경우, 임시 복구본에서 해당 정의만 제거해 읽는다.
  - 기존 병합 셀 범위를 열 삽입 후에도 유지한다.
  - `당초` 행은 원본 계산 의미를 유지하도록 같은 시트 행/열 참조를 보정한다.
  - `SUM(H26:H27)` 같은 범위 수식은 새로 삽입된 `변경` 행을 포함하지 않고 `당초` 행만 참조하도록 변환한다.
  - 다른 시트 참조 수식은 수정하지 않는다.
  - `변경` 행의 `금액` 열은 `노무비+재료비+경비` 수식으로 다시 설정한다.
  - Excel에서 열 때 전체 재계산되도록 저장 설정을 추가했다.

- `test_processor.py`
  - 샘플 workbook을 생성해 핵심 변환 로직을 검증한다.
  - `당초/변경` 표시, 빨간 글자 서식, 병합 셀 유지, 변경 행 금액 수식, 당초 행 수식 참조 보정을 확인한다.
  - 실제 `sample_*.xlsx` 파일이 있으면 스모크 테스트로 처리 가능 여부를 확인한다.

- `run_app.bat`
  - 로컬 번들 Python 경로로 GUI를 실행하는 Windows 런처를 추가했다.

- `process_sample.py`
  - 현재 작업 폴더의 `sample_*.xlsx` 파일을 직접 처리해 결과 파일을 재생성하는 보조 스크립트를 추가했다.

## 주요 문제와 해결

- `start "" ...` 실행 방식에서 Windows가 `\\` 경로를 찾는 오류가 발생했다.
  - `run_app.bat`로 실행 경로를 고정했다.

- 파일 선택 후 큰 엑셀 파일을 읽는 동안 GUI가 `응답 없음`으로 표시됐다.
  - 파일 읽기와 열 탐색을 백그라운드 스레드로 이동하고 상태 바를 추가했다.

- `당초` 행 수식이 원본과 다르게 계산되는 문제가 있었다.
  - 수식 문자열을 단순 보존하지 않고, 행/열 삽입 후 원본 행과 원본 열을 계속 참조하도록 같은 시트 참조를 보정했다.
  - 다른 시트 참조는 그대로 유지했다.
